Sistema de Control de Stock — Documentación Técnica

Descripción del sistema, estructura de archivos y diagrama de base de datos  ·  Versión 11  ·  Mayo 2026

1. Descripción General

El sistema de Control de Stock es una aplicación web que permite gestionar el inventario de medicamentos de la Farmacia Municipal de Cosquín. Funciona localmente en el servidor XAMPP de la PC de la farmacia y es accesible desde cualquier navegador de la red local (PC, notebook, tablet o celular).

Arquitectura: Aplicación web de dos capas — Frontend React (interfaz) ↔ API PHP (lógica y datos) ↔ Base de datos MariaDB (MySQL). Corre en XAMPP sobre Windows.

4. Explicación del Código Fuente

Backend PHP — Flujo de un request

GET /stock-control/api/medicamentos.php?id=5
        │
        ├─ helpers.php → startSession()     // inicia sesión PHP
        ├─ helpers.php → setCorsHeaders()   // permite requests desde React
        ├─ helpers.php → requireAuth()      // verifica $_SESSION['user'], devuelve 401 si no hay sesión
        │
        ├─ getDB()   → conexión PDO singleton a MariaDB
        ├─ getMethod() → 'GET'
        ├─ getId()     → 5
        │
        └─ match($method) → getMedicamento($db, 5)
                              └─ jsonResponse($data)  // header JSON + json_encode

Funciones clave en helpers.php

FunciónDescripción
startSession()Inicia la sesión PHP si no está activa
requireAuth()Verifica la sesión y devuelve 401 si el usuario no está logueado
setCorsHeaders()Agrega headers para permitir requests cross-origin desde el frontend
getDB()Devuelve una conexión PDO reutilizable a MySQL
jsonResponse($data)Envía la respuesta como JSON con el código HTTP correspondiente
jsonError($msg, $code)Envía un error en formato JSON
getBody()Lee y decodifica el cuerpo JSON del request (POST/PUT)
getId()Lee el parámetro ?id= de la URL como entero

Frontend React — Flujo de una pantalla

main.jsx         → monta BrowserRouter con basename="/stock-control"
   └─ App.jsx    → AuthProvider (sesión global) + sidebar + rutas protegidas
         └─ Medicamentos.jsx (página)
               ├─ useApi.js → useMedicamentos()  // llama a la API con axios
               ├─ useState / useEffect           // estado local y carga de datos
               ├─ tabla con datos del backend
               └─ <Modal> para altas y ediciones

Autenticación

Utiliza sesiones PHP ($_SESSION) en lugar de JWT porque Apache en Windows filtra el header Authorization: Bearer. El frontend usa withCredentials: true en axios para enviar la cookie de sesión en cada request. El contexto AuthContext.jsx mantiene el estado del usuario en React y redirige al login si la sesión expira.

Nota importante sobre el stock: products.stock es el valor autoritativo del stock total (se actualiza en cada operación). product_lots.quantity es la cantidad original del lote y nunca se decrementa. La distribución por ubicación se calcula en tiempo real desde stock_movements.

6. Proceso de Actualización del Sistema

1.  Desarrollador edita archivos fuente en:
      frontend/src/           (páginas React .jsx, hooks .js, estilos .css)
      xampp-deploy/api/       (endpoints .php)

2.  Compilar el frontend:
      cd frontend/
      npm run build           → genera frontend/dist/

3.  Copiar el build al deploy:
      dist/assets/* → xampp-deploy/stock-control/assets/
      dist/index.html → xampp-deploy/stock-control/index.html

4.  Empaquetar:
      Comprimir xampp-deploy/ → stock-control-vXX.zip

5.  Instalar en el servidor XAMPP:
      Extraer el ZIP en C:\xampp\htdocs\  (reemplaza archivos)

6.  Si hay cambios en la base de datos:
      Abrir en el navegador: http://localhost/stock-control/install.php
      (el instalador aplica ALTER TABLE de forma segura sin borrar datos)
